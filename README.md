# README для деплоя Chainlit (версия 0.2.0)

Этот проект содержит **автоматизированный скрипт деплоя** для развёртывания **Chainlit** (AI-чат с LLM) с полным стеком: PostgreSQL (pgvector), Ollama (LLM-сервер), History API и Nginx (HTTPS).  

Скрипт позволяет создавать **изолированные версии** проекта, каждая со своими томами, и управлять ими — деплоить, удалять, очищать старые.

## 🚀 Возможности

- **Полный стек**: Chainlit + PostgreSQL/vector + Ollama (с GPU) + History API + Nginx (HTTPS).
- **Автоматическая сборка** образа Chainlit локально (без попыток скачать с Docker Hub).
- **Уникальные тома** для каждой версии (данные PostgreSQL и Ollama изолированы).
- **Общие модели** Ollama (экономия места на диске).
- **Генерация** случайных секретов и `.env` для каждой версии.
- **Автоопределение порта транскрибатора** (если запущен `audio-transcriber-api`).
- **Освобождение занятого порта** (интерактивно или принудительно через `--force`).
- **Управление версиями**:
  - список (`--list`)
  - удаление конкретной версии (`--remove N`)
  - очистка старых версий с сохранением последних N (`--clean --keep N`)
- **Манифест** (`.deploy_manifest`) с информацией о ресурсах каждой версии.

## ⚙️ Требования

- **Docker** и **Docker Compose** (установлены).
- **Bash** (для скрипта).
- **openssl** (для генерации секретов и создания SSL-сертификатов).
- **NVIDIA GPU** (опционально, для Ollama) – если нет, уберите `deploy.resources` из compose.
- **Свободный порт** для HTTPS (по умолчанию 443) или готовность освободить занятый.
- **Предварительно скачанный дистрибутив Chainlit** – все исходные файлы Chainlit (app.py, модули, статика, конфигурация) должны быть размещены в папке-шаблоне `CHAINLIT_TEMPLATE/`. Скрипт копирует этот шаблон в каждую новую версию, поэтому необходимо заранее подготовить рабочую копию Chainlit с необходимыми кастомизациями.

> **Важно:**  
> - **Chainlit** (исходный код, модули, статика) копируется с хоста из папки шаблона при создании каждой новой версии. Убедитесь, что в `CHAINLIT_TEMPLATE/` находятся все необходимые файлы вашего проекта.  
> - **Модели Ollama** не копируются с хоста — они скачиваются автоматически через сервис `ollama-init` при первом запуске контейнеров (если их ещё нет в общем томе). Это позволяет экономить место и централизованно хранить модели для всех версий.

## 🔧 Настройка

1. **Убедитесь, что скрипт исполняемый**:
   ```bash
   chmod +x ~/dev/docker-containers/deploy.sh
   ```

2. **(Опционально) Настройте порт по умолчанию** в скрипте, изменив `DEFAULT_PORT="443"`, если нужно.

3. **Подготовьте шаблон** – скопируйте все файлы вашего проекта Chainlit в папку `CHAINLIT_TEMPLATE/`. Убедитесь, что там есть:
   - `app.py`
   - `modules/` со всеми модулями
   - `.chainlit/` с конфигурацией
   - `public/` с кастомизациями (если есть)
   - `docker-compose.yml`, `Dockerfile`, `nginx.conf` и другие необходимые файлы.

4. **Подготовьте сертификаты** (если используете HTTPS) – поместите `cert.pem` и `key.pem` в папку шаблона `CHAINLIT_TEMPLATE/certs/`. Скрипт скопирует их в `certs/` новой версии.

5. **Проверьте** наличие файлов шаблона (особенно `Dockerfile` и `docker-compose.yml`).

6. **Добавьте свои иконки и логотип** в папку `public/` шаблона (см. раздел [Мой вклад в разработку](#мой-вклад-в-разработку)).

### Создание SSL-сертификатов

Если у вас нет доверенных сертификатов, вы можете создать **самоподписанные** для тестирования. Для этого выполните команду в каталоге шаблона:

```bash
cd ~/dev/docker-containers/CHAINLIT_TEMPLATE/certs
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout key.pem -out cert.pem \
  -subj "/C=RU/ST=State/L=City/O=Organization/CN=your-domain.com"
```

**Важно**: 
- Замените `your-domain.com` на реальный домен или IP-адрес вашего сервера.
- Для production-среды используйте сертификаты от доверенного ЦС (например, Let's Encrypt).
- Сертификаты должны располагаться именно в папке `certs/` внутри шаблона, чтобы они копировались в каждую новую версию.

Скрипт автоматически скопирует их в новую версию, и Nginx будет использовать их для HTTPS.

### Генерация секретов

Для обеспечения безопасности скрипт автоматически генерирует случайные значения для переменных `CHAINLIT_AUTH_SECRET` и `POSTGRES_PASSWORD` при создании каждой новой версии. Эти секреты записываются в `.env` и используются сервисами.

Если вы хотите задать свои секреты (например, для согласованности между версиями), вы можете сгенерировать их вручную с помощью команды:

```bash
openssl rand -hex 32
```

Эта команда выдаст строку длиной 64 символа (например, `7a1b973756b3b08c7db06b57e40411993d2a3af802492bae940628e0ca027742`). Её нужно вставить в `.env` соответствующей версии в поля `CHAINLIT_AUTH_SECRET` и `POSTGRES_PASSWORD` соответственно.

> **Примечание:** Рекомендуется использовать разные секреты для разных версий и не хранить их в открытом виде в системе. Скрипт заботится об этом автоматически, но при ручной настройке следуйте этому же принципу.

## 📦 Использование

Все команды выполняются из каталога `~/dev/docker-containers`.

### 🟢 Деплой новой версии

```bash
./deploy.sh --port 443 --build
```

- `--port <порт>` – внешний HTTPS-порт (по умолчанию 443).  
- `--build` – принудительно пересобрать образ Chainlit (рекомендуется при первом деплое или после изменений в коде).  
- `--force` – автоматически освобождать занятый порт без подтверждения.

**Пример с автоматическим освобождением порта 443 (заменит текущий nginx):**
```bash
./deploy.sh --port 443 --build --force
```

После успешного деплоя:
- Контейнеры запущены, тома созданы.
- Сервис доступен по `https://<ваш-сервер>:<порт>`.
- В папке `CHAINLIT_DEV_<N>` созданы `.env`, `roles.json`, `.deploy_manifest`.

---

### 📋 Управление версиями

**Посмотреть все развёрнутые версии (с портами и датами):**
```bash
./deploy.sh --list
```

**Удалить конкретную версию (номер N):**
```bash
./deploy.sh --remove N
```
Эта команда останавливает контейнеры, удаляет тома, сеть, образы и папку проекта.

**Очистить старые версии, оставив последние K:**
```bash
./deploy.sh --clean --keep K
```
Например, оставить последние 2 версии:
```bash
./deploy.sh --clean --keep 2
```

---

### 🧹 Очистка системы

Если необходимо удалить **все следы** проекта (включая тома и папки), выполните:

```bash
# Удалить все папки версий
rm -rf ~/dev/docker-containers/CHAINLIT_DEV_*

# Удалить все тома
docker volume ls -q | grep chainlit-dev | xargs docker volume rm

# Удалить сети
docker network ls -q | grep chainlit-dev | xargs docker network rm

# Удалить образы chainlit-app
docker images -q --filter "reference=chainlit-app" | xargs docker rmi

# Очистить неиспользуемые ресурсы Docker
docker system prune -f
docker volume prune -f
```

## 🔐 Переменные окружения (.env)

Скрипт автоматически генерирует `.env` в папке новой версии со следующими параметрами:

| Переменная | Значение | Примечание |
|------------|----------|------------|
| `PROJECT_NAME` | `chainlit-dev-<N>` | Уникальное имя проекта |
| `NGINX_PORT` | выбранный порт | Внешний HTTPS-порт |
| `CHAINLIT_AUTH_SECRET` | случайная строка | Секрет для аутентификации (генерируется автоматически) |
| `POSTGRES_PASSWORD` | случайная строка | Пароль БД (генерируется автоматически) |
| `ASSISTANT_ROLE` | `assistant` | Роль ассистента (из `roles.json`) |
| `ROLES_FILE` | `roles.json` | Файл с описанием ролей |
| `AUDIO_TRANSCRIBER_API_URL` | `http://host.docker.internal:<ВАШ ПОРТ>/api/v1/transcribe` | URL транскрибатора (порт определяется автоматически) |

**Генерация секретов:** Скрипт автоматически создаёт случайные значения для `CHAINLIT_AUTH_SECRET` и `POSTGRES_PASSWORD` с помощью `openssl rand -hex 32`. Если вы хотите задать свои значения, сгенерируйте их такой же командой и замените в `.env` перед запуском compose (или после, но тогда потребуется перезапуск контейнеров).

Все переменные используются в `docker-compose.yml` через подстановку `${...}`.

*Микросервис транскрибатора развёрнут в отдельном контейнере и в данном readme не описан*

## ⚙️ Как это работает

1. Скрипт определяет номер следующей версии (`CHAINLIT_DEV_<N>`).
2. Копирует шаблон `CHAINLIT_TEMPLATE` в новую папку.
3. Копирует сертификаты (если есть).
4. Запрашивает или принимает порт.
5. Генерирует `.env` с уникальными значениями (включая случайные секреты).
6. Создаёт `roles.json` с дефолтной ролью ассистента (если файл отсутствует в шаблоне).
7. Собирает образ `chainlit-app:chainlit-dev-<N>` (если указан `--build` или образ отсутствует).
8. Запускает `docker compose up -d` (без `image:` – используется локальный образ).
9. При первом запуске (или если модели отсутствуют в общем томе) сервис `ollama-init` автоматически скачивает необходимые модели. Этот сервис выполняется один раз и после завершения останавливается (не перезапускается). Контейнер `ollama-init` **не удаляется**, но после успешного выполнения он находится в состоянии `Exited` и не влияет на работу остальных сервисов.
10. Создаёт манифест `.deploy_manifest` для отслеживания ресурсов.
11. Если указан `--clean` – удаляет старые версии.

## 🎨 Мой вклад в разработку

Данный проект является результатом **вайбкодинга** — всё опробовано на реальном сервере и работает стабильно. Основная кастомизация направлена на **изменение внешнего вида и поведения** Chainlit без изменения исходного кода фреймворка.

### Кастомизация UI

Вы можете настроить интерфейс через файлы:

- **`.chainlit/config.toml`** — основные настройки Chainlit (логотип, цветовая схема, поведение чата, системные подсказки).
- **`public/custom.css`** — пользовательские стили (переопределение стандартных классов, изменение шрифтов, цветов, расположения элементов).
- **`public/custom.js`** — пользовательские скрипты (добавление функциональности, например, автоматическое раскрытие подсказок, кастомные действия при отправке сообщения).

### Добавление брендинга

Для полной смены визуального стиля поместите в папку `public/` шаблона (и она автоматически скопируется в каждую новую версию) следующие файлы:

| Файл | Размер / формат | Назначение |
|------|-----------------|------------|
| `icon-192.png` | 192×192, PNG | Иконка для веб-приложения (favicon, PWA) |
| `icon-512.png` | 512×512, PNG | Иконка для больших экранов (PWA, splash screen) |
| `logo_1.svg` | SVG-вектор | Логотип, отображаемый в шапке чата и на странице входа |

**Примечание:** все файлы должны быть добавлены вручную. Если они отсутствуют, Chainlit использует стандартные иконки.

Все изменения в шаблоне автоматически применяются при следующем деплое новой версии.

## ⚠️ Известные ограничения

### Интеграция PostgreSQL (History API)

На данный момент реализована **базовая интеграция с PostgreSQL** для хранения истории чатов:

- ✅ **Создание** новых тредов — работает.
- ✅ **Чтение** тредов — работает.
- ✅ **Удаление** тредов — работает.
- ❌ **Редактирование** существующих тредов (переименование, изменение содержимого) — **не поддерживается**.

Это связано с тем, что текущая реализация History API (на основе Prisma) не предусматривает обновление уже созданных записей. Если вы хотите добавить возможность редактирования тредов, потребуется доработать:
- Модель данных в `prisma/schema.prisma` (добавить поля `updated_at` и разрешить обновление).
- API-эндпоинты в `history-api` (добавить `PUT` или `PATCH` методы).
- Интерфейс Chainlit (добавить кнопку редактирования названия треда).

Буду рад принять пул-реквесты с реализацией этой функции!

## ❗ Возможные проблемы

| Проблема | Решение |
|----------|---------|
| **Ошибка `pull access denied`** | Убедитесь, что в `docker-compose.yml` сервиса `chainlit` отсутствует строка `image:`. Образ должен собираться локально через `build: .`. |
| **Порт 443 занят** | Используйте `--port 8443` или остановите текущий nginx-контейнер (скрипт предложит). |
| **Не стартует база данных** | Проверьте логи: `docker compose -p chainlit-dev-<N> logs db`. Убедитесь, что порт 5432 не занят другим контейнером. |
| **Ollama не загружает модели** | Проверьте интернет-соединение и логи `ollama-init`. Если модель уже есть, можно закомментировать `ollama-init` в compose. Модели скачиваются автоматически при первом запуске в общий том (например, `chainlit-dev-ollama-models`), поэтому при создании новой версии они не скачиваются заново. |
| **Chainlit не видит транскрибатор** | Проверьте, что `audio-transcriber-api` запущен и доступен. Скрипт автоматически подставляет его порт, но если не определился – вручную укажите в `.env` `AUDIO_TRANSCRIBER_API_URL` с корректным портом (вместо `<ВАШ ПОРТ>`). |
| **Браузер не доверяет сертификату** | Используйте самоподписанный сертификат только для тестирования. Для production получите сертификат от Let's Encrypt или другого ЦС и поместите его в `certs/`. |
| **Отсутствуют файлы Chainlit в шаблоне** | Убедитесь, что папка `CHAINLIT_TEMPLATE/` содержит все необходимые файлы вашего проекта (app.py, modules, .chainlit и т.д.). Скрипт копирует их оттуда. |
